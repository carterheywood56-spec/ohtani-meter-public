# ohtani-meter-public ルール

## このディレクトリの役割

大谷翔平打撃状態メーターの **公開静的サイト**（手動更新前提）。
GitHub Pages 等で公開される 4 ページ構成：

| ファイル | 役割 |
|---|---|
| `index.html` | トップ・スコア概要・2×2 マトリクス |
| `method.html` | 指標の設計思想・計算フローの解説 |
| `case-study.html` | 過去事例（時系列ストーリー）|
| `faq.html` | よくある質問 |

全ファイル **完全静的**（fetch なし、数値ハードコード）。

## 編集方針

- **中身の差し替え**（数値・指標数・期間・文章）：このディレクトリで直接編集
- **デザインの刷新**（レイアウト・配色・構成変更）：`../ohtani-meter-design-lab/` で先に試作し確定後に反映
- スコア計算ロジック・基準値の正本は `../ohtani-meter/docs/decisions.md`。数値更新の前に本体スクリプトを実行して最新値を確認すること

## フロントエンド規約

- daisyUI コンポーネントを優先
- badge にテキストを入れる場合は `whitespace-nowrap` を付ける
- テーブルは `overflow-x-auto` でラップする
- 新規 CSS は最小限にする
- 紫グラデーション多用・過剰装飾は避ける

## Git 運用

- このディレクトリは独立した Git リポジトリ
- コミット・プッシュは明示的に依頼された場合のみ実行する（下記例外を除く）
- **例外**: `ohtani-meter/scripts/ohtani_daily.sh` 経由の `daily.html` 自動更新コミットは許可。ただし push はユーザー確認後に実施する
- 最終更新日は `index.html` の `<div class="hero-meta">` 内に手動記載
